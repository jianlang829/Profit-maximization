🧠 一次关于线性规划的朴素思考与验证

作者：jianlang

日期：2025年9月10日

“我不相信‘原料用尽就是最优’的假设，也不确定‘最优解一定在交点’——所以我决定自己写个程序，把所有可能的解都算一遍。（我开始不知道这是种什么方法，但觉得这样算出来好理解更对）”

🎯 项目背景

这是一个关于酒厂生产优化的简单线性规划问题：

  • 生产两种酒：低端酒（利润70元/瓶）和高端酒（利润120元/瓶）
  • 受两种原料限制：高纯度原料≤200kg，低纯度原料≤360kg
  • 目标：找到使总利润最大的生产组合 (x, y)

🤔 我的思考过程

面对这个问题，我首先对两种“标准解法”产生了质疑：

  1. 找交点法：为什么最优解一定在交点上？万一在边上或者框里面呢？
  2. 原料用尽假设：把原料用光就一定最赚钱吗？万一某种酒利润低还耗料多呢？

于是，我决定采用最朴素、最可靠的方法 —— 暴力枚举所有可能的整数解（当然后面知道这其实就是枚举）。

⚙️ 我的算法核心

根据约束条件，我推导出 x 的取值范围是 0 ≤ x ≤ 38（当然算错了，正确应该是0 ≤ x ≤ 45）。然后对每个 x，计算出满足约束的所有 y，并记录找出最大利润。

# 原始思考伪代码（来自 lrzd.txt）
条件：x≥0, y≥0
4x+7y≤200
8x+4y≤360
推导：0<=x<=38.5
目标：w=70x+120y 最大

算法：
for x in range(0, 38):
    # 对每个x，计算y的最大可行值
    y = min( (200-4*x)//7, (360-8*x)//4 )
    if y < 0: y = 0
    w = 70*x + 120*y
    # 比较并记录最大利润

✅ 验证结果 （代码在main.py）

通过暴力枚举，我得到了全局最优解：

生产 43 瓶低端酒 + 4 瓶高端酒 = 最大利润 3490 元

有趣的是，这个结果与“找交点法”的结果完全一致。这让我开始怀疑的相信了线性规划理论中“最优解在顶点”的定理 —— 后来知道它不是靠“相信”，而是靠亲手验证得来的。

💡 项目意义

这个小项目展示了：

  • 在数学和运筹学中，保持质疑精神的重要性。
  • 当理论让你困惑时，编程和计算是绝佳的验证工具。
  • “笨办法”（暴力枚举）在小规模问题中往往是最可靠的。
  • 理论和实践相互验证，才能建立真正的理解。
